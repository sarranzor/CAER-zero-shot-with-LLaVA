# Context-Aware Emotion Recognition (CAER)

Master's in Data Science thesis project at Universitat Oberta de Catalunya comparing two approaches to **context-aware emotion recognition (CAER)**: a dual-stream ResNet architecture with causal intervention (CCIM) and a zero-shot multimodal LLM (LLaVA-1.5 7B).

## Overview

Emotion recognition from images is traditionally solved using facial expression analysis alone. This project explores whether incorporating **scene context** alongside body/face information improves classification, and whether a large vision-language model can match or exceed trained discriminative models without any fine-tuning.

The experiments run on three datasets:

| Dataset | Classes | Description |
|---|---|---|
| **EMOTIC** | 6 | Real-world images; body bounding boxes provided; scene context used |
| **NCAER-S** | 7 | In-context facial expression dataset; face detected via Haar cascade |
| **SynthCAER** | 7 | Synthetic context-aware dataset; face detected via Haar cascade |

## Architecture

### Double ResNet + CCIM (`ccim_model.py`)

Two parallel ResNet-50 encoders process the scene context and the body/face crop independently:

- **ContextEncoder** — ResNet-50 pre-trained on Places365 (scene recognition)
- **BodyEncoder** — ResNet-50 pre-trained on ImageNet (body/face appearance)

Their 256-d projections are fused and passed through the **Causal Context Intervention Module (CCIM)** ([original paper](https://github.com/ydk122024/CCIM)), which deconfounds spurious correlations between scene context and emotion labels via a learnable confounder dictionary built with Mini-Batch K-Means.

### Baseline (`baseline_model.py`)

Same dual encoder + fusion, but with a standard linear classifier instead of CCIM. Used as the ablation upper bound.

### LLaVA-1.5 7B (`llava_shared.py`)

Zero-shot evaluation using `llava-hf/llava-1.5-7b-hf`. The model is loaded with NF4 4-bit quantization (bitsandbytes) when a GPU is available. Two prompting strategies are compared in ablation studies: face-only crop and full scene image.

## Setup

```bash
pip install -r requirements.txt
```

A CUDA-capable GPU is strongly recommended. LLaVA inference requires at least ~6 GB VRAM with 4-bit quantization; ResNet training requires ~4 GB VRAM with the default batch size of 32.

## Workflow

1. **Pre-process EMOTIC** — run `emotic_to_single-label.ipynb` to generate `emotic_basic.parquet`.
2. **Pre-process CAER to NCAER-S** — run `caer_to_ncaers.ipynb` to extract frames from the CAER videos and build `datasets/NCAERS/` along with `ncaers_index.parquet` (~28k images at one frame per 2-second segment).
3. **Train / evaluate ResNet models** — use the `resnet_baseline_*` and `ccim_*` notebooks in order.
4. **LLaVA zero-shot** — `llava_eval_all.ipynb` writes prediction parquets; `llava_ncaers_face_ablation.ipynb` compares prompting strategies. Pre-computed parquets are included so this step can be skipped.
5. **Run ablation studies** — `ncaers_ablation_study.ipynb` and `emotic_ablation_study.ipynb`.
